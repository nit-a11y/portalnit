# Guia de Migração para Supabase - Portal NIT

Este documento fornece todas as instruções necessárias para migrar o backend do Portal NIT de SQLite para o **Supabase** (PostgreSQL).

## 1. Esquema do Banco de Dados (SQL)

Execute o seguinte SQL no editor de consultas do Supabase para criar as tabelas necessárias:

```sql
-- Tabela de Solicitações (Requests)
CREATE TABLE requests (
  id BIGINT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
  ticket_id TEXT UNIQUE NOT NULL,
  name TEXT NOT NULL,
  department TEXT,
  role TEXT,
  email TEXT NOT NULL,
  phone TEXT,
  unit TEXT,
  project_name TEXT NOT NULL,
  request_type TEXT,
  current_problem TEXT,
  solution_objective TEXT,
  business_impact TEXT,
  problem_frequency TEXT,
  urgency TEXT,
  systems_involved JSONB, -- Armazenado como JSON para compatibilidade com arrays
  sectors_involved JSONB, -- Armazenado como JSON para compatibilidade com arrays
  description TEXT,
  status TEXT DEFAULT 'Recebido',
  created_at TIMESTAMPTZ DEFAULT NOW()
);

-- Tabela de Projetos (Projects)
CREATE TABLE projects (
  id BIGINT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
  title TEXT NOT NULL,
  tag TEXT,
  description TEXT,
  impact TEXT,
  image_url TEXT,
  created_at TIMESTAMPTZ DEFAULT NOW()
);

-- Habilitar Row Level Security (RLS) se necessário
-- Por padrão, você pode desabilitar RLS para desenvolvimento rápido ou criar políticas específicas.
ALTER TABLE requests ENABLE ROW LEVEL SECURITY;
ALTER TABLE projects ENABLE ROW LEVEL SECURITY;

-- Exemplo de política: Permitir leitura pública de projetos
CREATE POLICY "Permitir leitura pública de projetos" ON projects FOR SELECT USING (true);
```

## 2. Configuração do Cliente Supabase (Frontend)

Instale a dependência:
```bash
npm install @supabase/supabase-js
```

Crie um arquivo `src/lib/supabase.ts`:
```typescript
import { createClient } from '@supabase/supabase-js';

const supabaseUrl = import.meta.env.VITE_SUPABASE_URL;
const supabaseAnonKey = import.meta.env.VITE_SUPABASE_ANON_KEY;

export const supabase = createClient(supabaseUrl, supabaseAnonKey);
```

## 3. Substituição das APIs (Exemplos)

### Buscar Projetos (GET)
```typescript
const { data, error } = await supabase
  .from('projects')
  .select('*')
  .order('created_at', { ascending: false });
```

### Criar Solicitação (POST)
```typescript
const { data, error } = await supabase
  .from('requests')
  .insert([{
    ticket_id: 'NIT-2026-001',
    name: 'João Silva',
    // ... outros campos
    systems_involved: ['ERP', 'CRM'],
    sectors_involved: ['Comercial']
  }]);
```

### Gerenciar Projetos (Admin)
```typescript
// Inserir
await supabase.from('projects').insert([newProject]);

// Atualizar
await supabase.from('projects').update(updatedData).eq('id', id);

// Deletar
await supabase.from('projects').delete().eq('id', id);
```

## 4. Envio de E-mail (Edge Functions)

Como o Supabase é focado em banco de dados, o envio de e-mails (Nodemailer) deve ser movido para uma **Supabase Edge Function** ou você pode usar o serviço de e-mail integrado do Supabase (Auth) para notificações simples.

Para notificações complexas como as do NIT, recomenda-se usar uma Edge Function com um serviço como **Resend** ou **SendGrid**.

## 5. Variáveis de Ambiente (.env)

Adicione ao seu arquivo `.env`:
```env
VITE_SUPABASE_URL=https://seu-projeto.supabase.co
VITE_SUPABASE_ANON_KEY=sua-chave-anonima-aqui
```
